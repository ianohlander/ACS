## Design System Overview

### Core Visual Language

| Element | Specification |
|---------|---------------|
| **Primary Colors** | Deep forest green `#1F3D2B`, vibrant growth green `#2ECC71`, midnight blue `#0B1E2D` |
| **Accent Colors** | Electric cyan `#00D4FF`, warm gold `#FFD700` |
| **Shapes** | Organic curves, branching patterns, circular nodes, flowing connections |
| **Typography** | Modern sans-serif (primary), optional light serif (headings) |
| **Motion** | Subtle growth animations, pulsing connections, leaf-fall transitions |

### Brand Metaphors

| Term | Meaning |
|------|---------|
| **Seed** | Starting idea |
| **Roots** | Core systems / AI |
| **Branches** | Story paths / gameplay |
| **Leaves** | Content/assets |
| **Forest** | Collection of worlds |

### Image Generation Style Guide

All images should be generated with these style parameters:

- **Aspect ratio**: 16:9 for full screens, 4:3 for detail views
- **Quality**: High detail, 4K minimum
- **Lighting**: Dramatic but warm, emphasizing depth
- **UI style**: Clean, modern, with organic accents
- **Background**: Rich, layered environments

## Usage Guide

### Image Generation Tips

1. **Choose Your Tool**: These prompts work with Midjourney, DALL-E 3, Stable Diffusion XL, or similar AI image generators

2. **Select Priority Options**: Start with 1-2 options per screen that best match your target audience

3. **Iterate on Prompts**: 
   - Adjust colors to match brand palette
   - Modify UI elements to reflect actual functionality
   - Add/remove environmental details

4. **Generate Variants**: Create 2-3 variations of each promising concept

### Prompt Customization Formula

```
[Environment Description] + [UI Elements] + [Lighting/Mood] + [Style Tags]
```

**Example Customization**:

| Original | Customized |
|----------|------------|
| "Dark cavern" | "Dark cavern with purple accent lighting" |
| "UI panel" | "Floating glassmorphism UI panel" |
| "Mysterious" | "Mysterious, futuristic" |

### Recommended Workflow

```
1. Generate 3-5 concepts per screen
   ↓
2. Review and select top 2-3 per screen
   ↓
3. Create high-fidelity versions
   ↓
4. Test with target users
   ↓
5. Refine based on feedback
   ↓
6. Develop final UI system
```
