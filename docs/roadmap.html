ACS Project Roadmap

Adventure Construction Set Roadmap

This roadmap tracks the completed foundation work and the next planned milestones. The guiding idea is to evoke the original 1980s Adventure Construction Set while keeping rendering, editor UI, and simulation cleanly separated.

Strategic Direction

Vintage Feel

The project should capture the original ACS feeling through a classic gameplay panel, pixelated sprites, symbolic map icons, status rails, and a bottom message band.

Modern Architecture

The vintage look must remain a presentation mode over the same AdventurePackage, GameSessionState, and renderer-agnostic runtime-core.

Forward Milestone Path

10Next
Classic Renderer

Classic ACS Visual Mode

  • Add a classic-acs runtime presentation mode beside the current simple canvas renderer.
  • Render a fixed-aspect gameplay panel with map viewport, right status rail, and bottom message band.
  • Keep runtime-core renderer-agnostic so HD 2D and 3D remain future options.
  • Use pixelated scaling and a controlled palette to evoke the original era without copying the clunky old UI shell.
11Future
Assets

Sprite Manifests And Classic Asset Sets

  • Expand asset manifests for tiles, terrain, creatures, items, doors, portals, and UI indicators.
  • Add a first classic sprite set inspired by the collected reference images in legacy images/.
  • Map definitions to sprite IDs instead of hardcoded color assumptions.
12Future
Definitions

Definition Editors

  • Add editors for entity, item, terrain, and tile definitions.
  • Expose placement policy, sprite selection, stats, behavior profile, faction, dialogue reference, and possession data.
  • Keep placed instances separate from reusable definitions.
13Future
Rules

Thing, Trigger, And Text Editors

  • Add construction-set editors for interactive things, portals, events, map exits, messages, and dialogue records.
  • Build a visual rule editor over structured triggers/actions rather than arbitrary user code.
  • Support patterns like use item, defeat enemy, set flag, give item, change tile, move map, remove entity, show text, and complete quest.
14Future
World Structure

Map Scale And Adventure Structure Tools

  • Support map categories such as world, region, local, interior, and dungeon floor.
  • Add map creation/deletion, floor connections, region wiring, and map-specific palettes.
  • Prepare for adventures that mix overworld travel, encounters, interiors, and multi-floor dungeons.
15Future
Profiles

Character/Profile And Possession Systems

  • Add actor profile data inspired by original ACS sheets: life force, power, speed, skills, readied weapon, armor, and possessions.
  • Expose those fields in runtime status panels and editor definition screens.
  • Keep combat/profile rules in runtime-core or a future rules package, not in the renderer.

Architecture Principles

Renderer-Agnostic Engine

runtime-core owns commands, rules, state, events, triggers, and AI. Renderers consume state; they do not define gameplay rules.

Manifest-Based Assets

Content references stable asset IDs. Classic sprites, HD sprites, or 3D models can satisfy the same references later.

Editor-Core First

New editor capabilities should start as pure operations in editor-core, then be wired into browser UI.
