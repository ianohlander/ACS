ACS Version Trail

Version History

From Milestone 10 forward, each milestone records what changed, how to compare it, and which commit/tag represents that version.

Milestone 30Q: Shared Release Handoff Manifest

  • Added a typed ReleaseHandoffManifest so both forkable and standalone exports now carry the same release-level handoff summary.
  • Updated packaged handoff outputs so both artifact types now ship RELEASE-HANDOFF.json alongside their existing README and release-notes files.
  • Added Preview Release Handoff plus a dedicated Release Handoff Manifest card in Test & Publish.
  • Extended publishing, standalone-bundle, and editor UI smoke coverage so the shared release handoff preview and packaged manifest file are verified automatically.

Milestone 30P: Forkable Package Manifest Symmetry

  • Added a typed forkable package manifest so editable exports now carry an explicit file list and package entry file, not just handoff naming metadata.
  • Updated the forkable ZIP builder to package files from that shared manifest instead of re-deriving the package independently.
  • Updated the editor's forkable preview, readiness, and comparison text so the packaged file count and included file paths are visible before export.
  • Extended publishing unit coverage to verify the new forkable package manifest contents.

Milestone 30O: Forkable Project Package Export

  • Extended the forkable handoff manifest with recommended ZIP/extracted-folder names plus packaged README and RELEASE-NOTES.txt paths.
  • Added packages/publishing/src/forkable-package.ts so forkable exports now package forkable-project.json, project-manifest.json, README files, and release notes into a ZIP archive.
  • Updated the editor export flow so forkable handoffs now download as a packaged ZIP instead of a raw JSON file, while preview/readiness/comparison panels surface the packaged file names.
  • Extended publishing unit coverage to verify the packaged forkable archive contents.

Milestone 30N: Handoff Naming And Release Notes Packaging

  • Extended the standalone distribution manifest with a recommended archive file name, extracted-folder name, and packaged RELEASE-NOTES.txt path.
  • Updated the standalone bundle builder so exported packages now include RELEASE-NOTES.txt and mention it in the packaged README files.
  • Updated the editor download flow so forkable and standalone exports use manifest-backed handoff file names instead of ad hoc naming.
  • Extended preview/readiness summaries and unit coverage so the final shipped handoff names are visible and verified before export.

Milestone 30M: Artifact Comparison Panel

  • Added an Artifact Comparison card in Test & Publish.
  • Updated the editor to compare forkable and standalone artifacts side by side, including editable-versus-play-only purpose, handoff details, and whether both previews come from the same release.
  • Extended the editor UI smoke test so the comparison panel must render meaningful status text and list content.
  • Folded the future guided forkable-import planning into the roadmap and LLM-readable project context.

Milestone 30L: Forkable Project Handoff Manifest

  • Added a typed ForkableProjectManifest to forkable publishing artifacts so editable export identity, release metadata, import guidance, and known limitations travel with the artifact.
  • Updated the local API export flow so forkable artifacts are populated from immutable release metadata including release id, label, version, and release notes.
  • Updated the editor's forkable preview and release readiness summaries to show the manifest-backed recommended file name, import area, handoff next steps, and known limitations.
  • Extended publishing unit coverage to verify the new manifest and release-backed handoff details.

Milestone 30K: Forkable Artifact Preview

  • Added a Preview Forkable Artifact action and a dedicated forkable preview card in Test & Publish.
  • Updated the editor to request the latest release-backed forkableProject artifact and render editable handoff facts such as starter-pack preservation and custom-library counts.
  • Extended Release Readiness so the publishing checklist now reflects whether the editable export path has been previewed too.
  • Extended the editor UI smoke test to verify the new preview button, disabled state, and preview panel.

Milestone 30J: Standalone Handoff Guide

  • Extended the standalone distribution manifest with packaged handoff-guide metadata including README file paths and the recommended launch path.
  • Added README.html and README.txt to standalone bundles so recipients can see how to launch the package locally or host it as a static web bundle.
  • Updated standalone preview and release readiness summaries so the editor surfaces those packaged handoff guides before export.
  • Extended publishing and standalone-bundle tests to verify the new metadata and bundled instruction files.

Milestone 30I: Local Launcher Groundwork

  • Extended the standalone distribution manifest with bundled local-launch metadata such as helper-script paths and default port.
  • Added packaged Windows launcher helpers at launch/run-local.ps1 and launch/run-local.cmd so exported games can be served locally without manually composing a server command.
  • Kept the standalone package fundamentally a static web bundle while making local launcher support a convenience layer over that same bundle.
  • Updated standalone preview and release readiness summaries so the editor shows bundled local-launch availability.

Milestone 30H: Standalone Distribution Manifest

  • Added a typed StandaloneDistributionManifest to standalone publishing artifacts so release identity, entry file, content counts, and known limitations travel with the package.
  • Updated the local API export flow so standalone artifacts are populated from immutable release metadata including release id, label, version, and release notes.
  • Added bundle/distribution-manifest.json to packaged standalone bundles.
  • Updated standalone package preview and release readiness summaries to report distribution-manifest details instead of only raw bundle facts.

Milestone 30G: Release Notes And Metadata

  • Added release metadata support so immutable release records can persist a human-readable label plus releaseNotes.
  • Added a Release Notes card in Test & Publish with fields for the next release label and notes.
  • Updated release summaries and the Release Readiness checklist so published builds show note excerpts and whether shipping notes exist.
  • Updated the editor UI smoke test and project-api client unit coverage for the new publish metadata flow.

Milestone 30F: Release Readiness And Known Limitations

  • Added a Release Readiness card in Test & Publish.
  • Derived readiness from validation state, published release presence, standalone package preview state, diagnostics availability, and current MVP known limitations.
  • Surfaced current constraints directly in the editor so sharing readiness is easier to judge without cross-referencing roadmap notes.
  • Refactored project action-button state to stay under the complexity gate while extending the packaging workflow.

Milestone 30E: Standalone Package Preview

  • Added a Preview Standalone Package action and a standalone package preview card in Test & Publish.
  • Updated the editor to request the release-backed standalonePlayable artifact on demand and render bundle entry file, asset counts, and shipped file paths.
  • Invalidate standalone preview state whenever the draft changes so stale release contents are not shown against a modified draft.
  • Extended the editor UI smoke test to cover the preview button and preview panel.

Milestone 30D: Standalone ZIP Export

  • Added packages/publishing/src/standalone-archive.ts to turn standalone bundle manifests into valid ZIP archives.
  • Updated the editor export flow so standalone artifacts download as .zip while the forkable export continued using the release-backed artifact path until later packaging work.
  • Renamed the editor action to Export Standalone ZIP and added the missing browser import-map entry for @acs/publishing.
  • Added unit coverage proving the generated standalone bundle archive contains expected packaged files.

Milestone 30C: Standalone Play Bundle Foundation

  • Extended standalonePlayable artifacts with bundle-manifest records for file path, content type, and generated contents.
  • Added apps/api/src/standalone-bundle.ts to assemble a static play-shell manifest with index.html, runtime assets, and bundle/adventure-package.json.
  • Updated the release artifact API path so standalone exports attach the generated bundle manifest automatically.
  • Updated the browser runtime entry so apps/web/src/index.ts can boot directly from ?package=... for standalone play bundles.
  • Added publishing validation coverage for attached bundle manifests and a unit test for generated standalone bundle contents.

Milestone 30B: Release Export Workflow

  • Added release artifact export routes to the local API.
  • Added typed exportReleaseArtifact(...) support to @acs/project-api.

Milestone 30A: Publishing Artifact Foundation
