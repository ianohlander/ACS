ACS Version Trail

Version History

From Milestone 10 forward, each milestone should record what changed, how to compare it, and which commit/tag represents that version.

Milestone 13: Dialogue And Trigger Editors

Status: complete after commit and tag creation.

Git tag: milestone-13

Primary comparison target: compare milestone-12 with milestone-13 to see the editor gain dialogue text editing and structured trigger editing.

  • Added editor-core.updateDialogueNode and editor-core.updateTriggerDefinition for pure package-level edits.
  • Added editor sidebar panels for existing dialogue records and trigger records.
  • Kept trigger conditions/actions as structured package data, not executable scripts.
  • Changed editor startup so the local draft renders before the optional API connection completes.
  • Updated app labels, roadmap, user guide, system reference, guide graphics, and PDFs for Milestone 13.

Milestone 11: Sprite Manifests And Classic Asset Sets

Status: complete after commit and tag creation.

Git tag: milestone-11

Primary comparison target: compare milestone-10 with milestone-11 to see the classic renderer move from hardcoded drawing assumptions to manifest-driven sprite lookup.

  • Added VisualManifestDefinition and ClassicSpriteStyle content types.
  • Added the sample adventure's Classic Solar Seal Sprite Set.
  • Mapped Hero, Oracle, and Shrine Wolf definitions to stable sprite IDs.
  • Updated runtime-2d classic mode to resolve tile and entity visuals through the manifest.
  • Added validation warnings for missing classic tile/entity sprite entries.
  • Updated app labels, roadmap, user guide, system reference, and PDFs for Milestone 11.

Milestone 12: Entity Definition Editor

Status: complete after commit and tag creation.

Git tag: milestone-12

Primary comparison target: compare milestone-11 with milestone-12 to see the editor gain reusable entity definition editing.

  • Added editor-core.updateEntityDefinition for pure package-level definition edits.
  • Added an editor sidebar panel for entity definition name, kind, placement, sprite asset ID, faction, and behavior tuning.
  • Kept reusable definitions separate from placed entity instances.
  • Updated app/editor labels, roadmap, user guide, system reference, and PDFs for Milestone 12.

Milestone 10: Classic ACS Visual Mode

Status: complete after commit and tag creation.

Git tag: milestone-10

Primary comparison target: compare milestone-10 with future milestone tags.

  • Added classic-acs and debug-grid runtime visual modes in runtime-2d.
  • Defaulted the browser runtime to the classic ACS-inspired presentation.
  • Added a runtime Visual Mode selector that switches presentation without changing the engine session.
  • Rendered a vintage-style play panel with a map viewport, right-side status rail, and bottom message band.
  • Kept runtime-core unchanged so gameplay logic remains renderer-agnostic.
  • Updated the user guide, system reference, roadmap, and runtime guide graphic.
