ACS Version Trail

Version History

From Milestone 10 forward, each milestone records what changed, how to compare it, and which commit/tag represents that version.

Milestone 30C: Standalone Play Bundle Foundation

  • Extended standalonePlayable artifacts with bundle-manifest records for file path, content type, and generated contents.
  • Added apps/api/src/standalone-bundle.ts to assemble a static play-shell manifest with index.html, runtime assets, and bundle/adventure-package.json.
  • Updated the release artifact API path so standalone exports attach the generated bundle manifest automatically.
  • Updated the browser runtime entry so apps/web/src/index.ts can boot directly from ?package=... for standalone play bundles.
  • Added publishing validation coverage for attached bundle manifests and a unit test for generated standalone bundle contents.

Milestone 30B: Release Export Workflow

  • Added release artifact export routes to the local API.
  • Added typed exportReleaseArtifact(...) support to @acs/project-api.
  • Added Export Forkable JSON and Export Standalone JSON controls to the editor's Test & Publish panel.
  • Added a client unit test and editor UI smoke assertions for the new export workflow.

Milestone 30A: Publishing Artifact Foundation

  • Added a new @acs/publishing package for pure export artifact generation.
  • Added typed forkableProject and standalonePlayable artifact builders.
  • Added runtime asset dependency collection, standalone authoring-data pruning, and publish-artifact validation helpers.
  • Added unit coverage proving forkable exports preserve editable data, standalone exports strip starter-library packs, and missing runtime assets are reported before acceptance.

Milestone 27: Editor Information Architecture And Rename Tools

  • Added a selected-cell inspector that summarizes tile, occupant, exit, and trigger data for the clicked map coordinate.
  • Added UI-based Display Rename / Reskin preview and apply controls that change player-facing text while preserving stable ids and structured references.
  • Added shared editor-core display rename preview/apply helpers for future API, undo, validation, and import/export reuse.
  • Updated the Relay Station Alecto tutorial with real screenshots for selected-cell inspection and display rename preview.

Milestone 26: Starter And Custom Library Portability Foundation

  • Added @acs/default-content as the boundary for built-in starter library metadata and custom library export helpers.
  • Documented the storage split between built-in starter libraries, adventure-local objects, and reusable custom library exports.

Milestone 25: Authoring Diagnostics And Test Harness

  • Added reusable editor-core authoring diagnostics for triggers, entities, exits, flags, inventory, quests, and scenario prompts.
  • Added compact Authoring Diagnostics and Playtest Scenarios cards in the Test & Publish workspace.
  • Added npm run playtest:smoke to build, validate, and simulate Oracle interaction, shrine reward, item grant, tile change, quest stage, and exit travel flows.
  • Updated the roadmap, User Guide, System Reference, and AI-readable project context for the milestone.

Milestone 24: Presentation, Pixel Art, And Starter Libraries

  • Added splash-screen, starting music, intro text, manifest-backed pixel sprites, and Assets-panel pixel painting.
  • Expanded stocked starter libraries with genre packs and ACS-inspired generic objects.

Milestone 23: Object-Backed Quest Objectives

  • Converted quest stage text into nested objective objects with ids, kinds, descriptions, and optional map/item targets.
  • Converted reward notes into nested reward objects with ids, kinds, labels, and optional item references.
  • Moved quest editor-core operations into quest-definitions.ts as the first concrete repo-wide organization cleanup slice.
  • Updated the current Libraries/Quests screenshot, User Guide, and System Reference to describe the new object-backed workflow.

Milestone 22: Quest And Objective Builder

  • Added quest definition creation/editing in the Libraries panel.
  • Added setQuestStage trigger actions, runtime questStageSet events, validation checks, and Objective panel rendering from quest state.
  • Updated current screenshots and documentation to showcase quest stages and trigger-stage wiring.
