ACS Version Trail

Version History

From Milestone 10 forward, each milestone records what changed, how to compare it, and which commit/tag represents that version.

Milestone 31L: Portable AI Import Dossiers

  • Extended @acs/ai-core with AiGenerationSessionImportDossier, AiGenerationSessionImportDossierManifest, createGenerationSessionImportDossier(...), and validateGenerationSessionImportDossier(...).
  • The new import dossier bundles the shared handoff report, import plan, import report, required dossier file list, and a human-readable README into one provider-agnostic import-review object.
  • Extended unit coverage to verify both ready and mismatched import-dossier cases.
  • Refreshed roadmap, references, LLM context, and PDFs for the new Milestone 31 portable import-dossier layer.

May 2026: Variable Sprite Scale Planning

  • Reviewed the ADR 0004 variable-sprite-scale documents and accepted the capability as future renderer-family work rather than a classic-mode gameplay rewrite.
  • Placed optional entity/tile visual scales, render layers, higher-resolution visual-manifest records, and any future pixel-accurate collision or missile-distance helpers into Milestone 38.
  • Placed the higher-resolution asset-authoring follow-through, including HD sprite records, scale-aware previews, optional collision-mask authoring, and validation into Milestone 39.
  • Updated the roadmap, architecture, System Reference, User Guide, testing strategy, and LLM project context so classic 8-bit behavior remains unchanged while richer renderer families gain a clear future path.

Milestone 31K: Shared AI Handoff Import Reports

  • Extended @acs/ai-core with AiGenerationSessionImportReport and createGenerationSessionImportReport(...) so later editor/API layers can explain whether a reviewed AI handoff is structurally ready to ingest.
  • The import report summarizes import status, blocker counts, required files, preserved artifacts, summary lines, and one next-step recommendation from the reviewed handoff state.
  • Extended unit coverage to verify both ready and blocked AI handoff import-report cases.
  • Refreshed roadmap, references, LLM context, and PDFs for the new Milestone 31 import-report layer, while also adding the durable UX skinning inventory and retained world-structure summary.

May 2026: UX Skinning Inventory

  • Added docs/ux-skinning-inventory.md as the human-readable registry of live Edit Mode and Play Mode UI surfaces, shared primitives, dynamic elements, and future skin skeleton requirements.
  • Added docs/ux-skinning-inventory.json as the machine-readable inventory for future milestone reviews and skin-package planning.
  • Updated roadmap, architecture, System Reference, User Guide, testing strategy, and LLM project context so UI-changing milestones must keep the skinning inventory current.

Milestone 31I: Shared AI Handoff Integrity Reports

  • The handoff report deduplicates validation issues, summarizes package/bundle/archive status, lists required files and archive entries, and provides one next-step recommendation for export, import, or apply review.
  • Extended unit coverage to verify both clean handoff reports and blocked archive-parity scenarios.
  • Expanded the roadmap so Milestones 32 through 39 are all visible with statuses and intended outcomes, then refreshed references, LLM context, and PDFs for the new Milestone 31 integrity layer.

Milestone 31H: Archive-Ready Review Handoffs

  • Extended @acs/ai-core with AiGenerationSessionPackageArchive, createGenerationSessionPackageArchive(...), and validateGenerationSessionPackageArchive(...) so later editor/API layers can hand off one real archive artifact instead of rebuilding ZIP packaging from raw file lists.
  • The AI review handoff now carries archive bytes plus declared archive entry metadata for the package manifest, session record, change summary, application plan, and README files.
  • Archive validation now checks archive name, extracted-folder name, entry presence, entry media types, entry sizes, and non-empty archive payloads against the generated package bundle.
  • Extended unit coverage and refreshed roadmap, references, LLM context, and PDFs for the new Milestone 31 archive layer, while also folding accepted world-structure editor and architecture planning into the roadmap memory.

Milestone 31G: Export-Ready Review Bundles

  • Bundle validation now checks archive-name, extracted-folder-name, and required-file parity against the package manifest before later export or import flows rely on the bundle.
  • Extended unit coverage and refreshed roadmap, references, LLM context, and PDFs for the new Milestone 31 export-bundle layer.

Milestone 31F: Portable Review Packages

Milestone 31E: Proposal Application Plans

Milestone 31D: Proposal Change Summaries
